Random Walk Simulation of Server Security

Introduction

This simulation models a server that receives weekly security updates over n weeks, while facing m attackers capable of breaching it with probability \( p \).

Each week, a weekly score \( X_i \) is assigned. Its values are:

\[ X_i = \begin{cases} +1, & \text{if the server remains secure in week i} \\ -1, & \text{if the server is breached in week i} \end{cases} \]

The cumulative score after \( n \) weeks is given by:

\[ S_n = \sum_{i=1}^{n} X_i \]

To connect this to a binomial variable, let \( B \) denote the number of weeks in which the server remains secure. Since each week is independent and a "secure" week occurs with probability \( 1-p \), \[ B \sim \mathrm{Binomial}(n, 1-p). \]

There are \(B\) weeks with score \(+1\) and \(n-B\) weeks with score \(-1\). Therefore:

\[ S_n = (+1)\cdot B + (-1)\cdot (n - B) = B - (n - B) = 2B - n. \]

Substituting the distribution of \( B \) gives the relationship \[ S_n = 2\,B(n,1-p) - n, \] i.e. the distribution of \(S_n\) is a linear transformation of a binomial random variable (a rescaling by 2 and a shift by \(-n\)).

As \( n \) and the number of simulated trajectories \( m \) grow, the empirical frequencies of \( S_n \) approach the theoretical binomial probabilities. This illustrates the convergence of the random walk model to the binomial distribution.
