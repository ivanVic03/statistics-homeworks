Theoretical Analysis of the Counting Process

The Approximated Stochastic Process

The simulation analyzes the Poisson Limit Theorem, which states that a Binomial distribution, $Bin(n, p)$, converges to a Poisson distribution as $n \to \infty$ (the number of trials) and $p \to 0$ (the probability of success) such that the mean $\lambda = n \times p$ remains constant.

Our simulation is a $Binomial(n, \lambda/n)$ process. Here, we have:

  • A large number of subintervals, $n$.
  • A small probability of success in each interval, $p_{sub} = \lambda/n$.
  • A constant mean, $n \times p_{sub} = n \times (\lambda/n) = \lambda$.

This process is a discrete-time approximation of the Homogeneous Poisson Process (HPP). The HPP is a continuous-time stochastic process that counts the number of "events" (or "successes") that occur in a given time interval.

The final count $K$ generated by our $Binomial(n, \lambda/n)$ simulation is a single random draw from a distribution that approximates $Poisson(\lambda)$. The graph is a realization of this discrete-time approximation.

Theoretical Properties

The Poisson Process $N(t)$, which our simulation approximates, is defined by the following theoretical properties:

  1. Counting Starts at Zero: The count of events at time $t=0$ is zero.
    \( N(0) = 0 \)
  2. Independent Increments: The number of events in non-overlapping time intervals are independent random variables.
    (e.g., the number of events in $[0, 2]$ is independent of the number of events in $[3, 5]$).
  3. Stationary Increments: The probability distribution of the number of events in any interval depends only on the length of the interval, not its location.
    \( P(N(t+s) - N(t) = k) \) depends only on $s$.
  4. Poisson Distribution: The number of events in any interval of length $t$ follows a Poisson distribution with parameter $\lambda t$.
    \( P(N(t) = k) = \frac{e^{-\lambda t} (\lambda t)^k}{k!} \)
  5. Rarity of Events: In a very small time interval $\Delta t$, the probability of more than one event occurring is negligible (goes to zero faster than $\Delta t$). The probability of one event is approximately $\lambda \Delta t$.
    \( P(N(t+\Delta t) - N(t) = 1) \approx \lambda \Delta t \)

Our $Binomial(n, \lambda/n)$ simulation explicitly models this last property. Our small interval is $\Delta t = 1/n$, so the probability of one event is exactly $p_{sub} = \lambda/n = \lambda \Delta t$.

Interpretation of the Rate Parameter λ

The parameter $\lambda$ (lambda) is the average rate or intensity of the process.

Its interpretation is "the expected number of events per unit of time." In our simulation, the total time interval is $T=1$, so $\lambda$ is simply:

$\lambda$ = The expected total number of events in the entire interval $[0, 1]$.

If our final count $K=5$, and the calculated $\lambda=4.8$, this is consistent. $\lambda$ is not the number of events that will happen (that is $K$), but the average number of events we would expect to see if we ran the simulation many, many times.

In the context of the Poisson distribution $Poisson(\lambda)$, the parameter $\lambda$ is both the mean and the variance of the distribution. This means that if we were to run our $Binomial(n, \lambda/n)$ simulation many times and plot a histogram of the final $K$ counts, that histogram would approximate a Poisson distribution with:

  • Mean \( E[K] = \lambda \)
  • Variance \( Var(K) = \lambda \)

Therefore, $\lambda$ is the single most important parameter, as it defines the central tendency and the spread of the resulting count distribution.
